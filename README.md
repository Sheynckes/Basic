## Python简介

C语言开发效率太低，而shell只是一种胶水语言。Python的作者[Guido van Rossum](https://en.wikipedia.org/wiki/Guido_van_Rossum)希望创造一种C和shell之间，功能全面，易学易用，可扩展的语言。

- **简单，容易学习。**
- **免费开源。**
- **高级语言，细节隐藏**：尤其是对于指针和内存的管理。Java虽然号称是高级语言、隐藏了所有细节，但是如果用Java写底层服务，比如写一个分布式的服务系统，仍然会存在空指针、内存泄露等问题；甚至垃圾处理的实现细节有时也会对程序员造成很大的困扰。Python就不存在这些问题。
- **解释性，可移植；一次编写，到处运行。**相比而言，C语言的跨平台移植就会存在很大的问题。
- **面向对象。**
- **可扩展；可嵌入C/C++，或者使用C/C++提高性能。**虽然Python性能不好，但可以使用很多方法来提升性能，比如使用Jython（底层使用Java虚拟机）来提高代码执行效率、使用C++对耗时的算法进行扩展来提高代码执行速率等。
- **扩展库丰富。**


- **慢。**
- **shell脚本不友好。**

## 如何学好编程

- 基础部分
  - 基本语法：基本数据类型、对象的定义和声明、循环、判断、逻辑运算、位运算
  - 数据结构：**数组**、**字符串**、系统标准库自带的数据类型、如何实现常用的数据结构（链表、堆栈、二叉树、……）

  - 异常处理：抛出合捕获异常（try/catch，try/except）、异常和错误的区别以及应用场合；
- 进阶提高
  - 常用算法：分治、贪心、动态规划；
  - 数据库访问：建立连接、执行SQL查询、读取查询记录；
  - 面向对象：继承、多态、静态变量与方法；
  - 其他：多线程/进程、匿名函数、语言相关特定知识（Java的发射、C++的模板、Python的协程、……）

## Python 2.x与3.x的区别

- Unicode：3.x默认使用unicode编码
- 除法的区别
- 异常：3.x只能抛出继承自BaseException的异常
- xrange：3.x取消了xrange，range和xrange一样实现为惰性求值
- 二/八进制：3.x必须强制写成0b1001和0o7236
- 不等于：3.x中取消了`<>`，只保留`!=`
- 模块名称更改：3.x中Queue改为queue、repr改为reprlib、……
- 3.x中取消了long，统一为int
- 3.x中新增了bytes类型（可理解为二进制的字符串），并可与string相互转换
- 3.x中dict的keys/items/values方法返回的是一个迭代器，iterkeys等函数被废弃，has_key被in取代。
